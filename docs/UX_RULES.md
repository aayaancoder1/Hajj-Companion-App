# UX Rules

## Core UX Philosophy

The app should feel like:

> “a calm companion during Hajj”

NOT:

> “a feature-heavy dashboard”.

Every screen should reduce:

* confusion
* stress
* uncertainty
* cognitive overload

---

# Visual Direction

The UI should feel:

* emotionally warm
* spacious
* grounded
* modern
* trustworthy
* spiritually respectful

Avoid:

* corporate SaaS aesthetics
* crypto/web3 design
* excessive gradients
* flashy animations
* neon colors
* cluttered layouts

---

# Interaction Philosophy

Interactions should feel:

* smooth
* intentional
* calm
* lightweight

Avoid:

* overcomplicated gestures
* hidden navigation
* confusing flows
* excessive modal stacking

---

# Layout Rules

* Mobile-first always
* Use consistent spacing
* Use reusable cards/components
* Preserve clean visual hierarchy
* Prioritize readability and clarity

---

# Navigation Rules

Primary navigation:

* bottom navigation bar
* persistent across main screens

Main routes:

1. Home
2. Haram Guide
3. Rituals
4. Nearby
5. Emergency

---

# Component Rules

Always prefer:

* reusable components
* shared UI primitives
* modular architecture

Avoid:

* duplicate components
* page-specific styling duplication
* giant monolithic files

---

# Emotional Tone

The app should feel:

* reassuring
* dependable
* calm
* thoughtful

Microcopy should avoid:

* hype
* exaggerated “AI” language
* robotic phrasing

---

# AI Assistance Button

The floating Gemini/Google AI button:

* is a fallback helper
* not an in-app chatbot
* not conversational UI

It should feel:

* subtle
* useful
* integrated

NOT:

* flashy
* intrusive
* attention-seeking
